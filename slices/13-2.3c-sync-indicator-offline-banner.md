SLICE <13-§2.3c> — Icon-only sync indicator + offline banner (no badge while synced)

BUDGET

  eating the budget, ship the `navigator.onLine` + fetch-failure heuristic in SCOPE

GOAL
  Three Owner-requested UX changes to how sync state is shown, replacing the current
  always-visible text badge: (1) nothing is shown while synced/online — no badge, no
  text; (2) a persistent thin banner reading "Офлайн" appears at the top **only**
  when the device is actually offline; (3) a genuinely pending or failed mutation is
  shown as a small **icon**, never text.

SCOPE
  1. **Connectivity store** — add `app/src/store/connectivity.ts` (Zustand), state
     `{ isOnline: boolean }` initialized from `navigator.onLine`. Wire:
     - `window.addEventListener('online', ...)` / `('offline', ...)` to set it.
     - In `app/src/api/session.ts`'s `fetchAuth()`: if the `fetch()` call itself
       throws (a `TypeError` — network failure / DNS / no connection — as opposed to
       a normal HTTP error response, which still means the server *was* reached),
       set `isOnline: false`; on any request that **completes** (any HTTP status,
       even 4xx/5xx — reaching the server at all proves connectivity), set
       `isOnline: true`. This catches "device has Wi-Fi but can't reach the server"
       in addition to the browser-level online/offline events, which `navigator.onLine`
       alone does not.
  2. **`OfflineBanner` component** — `app/src/components/OfflineBanner.tsx`. Port the
     existing design reference verbatim: `design/screens2.jsx` line 72
     (`<div className="offline-banner">{Ic.wifiOff}Офлайн · отметките се запазват и
     ще се синхронизират</div>`) and its CSS from `design/app.css` lines 248-254
     (`.offline-banner` — not yet ported into `app/src/list-screens.css`; port it
     now, scoped under `.si-root` like the rest of that file). Port the `wifiOff` SVG
     icon from `design/ui.jsx` line 30 into the component (or a small shared
     `app/src/components/icons.tsx` if that's cleaner — your call). Render this
     component **once**, fixed at the top of the screen, from `app/src/App.tsx` —
     not per-screen — visible whenever `useConnectivityStore` reports
     `isOnline: false`, hidden otherwise.
  3. **Split pending vs. failed in the mutation-count selector.** In
     `app/src/storage/db.ts`, replace `getPendingMutationCounts()` (lines ~190-203)
     with `getMutationStatusCounts(): Promise<Record<string, { pending: number;
     failed: number }>>` — `pending` covers `status ∈ {pending, in_flight}`,
     `failed` covers `status === 'failed'`, both keyed by `entity_client_uuid` as
     before. Update the one caller (`HomeScreen.tsx`'s `refreshPendingCounts`) and
     its state/prop name accordingly (e.g. `mutationStatusCounts`).
  4. **New shared `SyncStatusIndicator`** — `app/src/components/SyncStatusIndicator.tsx`,
     replacing the two inline duplicates (`ListScreen.tsx` lines ~22-27,
     `ListsScreen.tsx` lines ~18-23). Props: `{ pending: number; failed: number }`.
     Render rule:
     - `failed > 0` → render a small warning-triangle icon (new SVG, same stroke
       style as `design/ui.jsx`'s `Ic` set — e.g. viewBox 24x24, `stroke="currentColor"`,
       `strokeWidth ~1.8`) with `aria-label="Грешка при синхронизация"` / `title`
       attribute for accessibility (icon-only visually, but never icon-*only* for a
       screen reader). **Always rendered when `failed > 0`, even while online** — per
       `10-ux-rules.md` §7, a real rejection is never hidden.
     - else `pending > 0` → render a small pending/syncing icon (new SVG, same
       style — e.g. two curved arrows forming a refresh/sync glyph) with
       `aria-label="Синхронизира се"`.
     - else (both zero) → render `null`. No dot, no "synced" text, nothing in the
       DOM. This is the common case while online.
  5. Wire the new `SyncStatusIndicator` into `ListScreen.tsx` and `ListsScreen.tsx`
     in place of their inline versions, passing `{ pending, failed }` from the
     renamed `mutationStatusCounts` prop instead of the old flat `pendingCounts`
     number.

TESTS (Codex writes and greens these)
  - `SyncStatusIndicator`: renders nothing for `{pending: 0, failed: 0}`; renders the
    pending icon (by `aria-label`) for `{pending: 1, failed: 0}`; renders the failed
    icon for `{pending: 0, failed: 1}` and also when both are non-zero (failed wins).
  - `OfflineBanner`: hidden when the connectivity store reports online; visible with
    the exact Bulgarian copy when it reports offline.
  - `connectivity.ts`: a simulated `fetch` network-level throw flips the store to
    `isOnline: false`; a subsequent successful (even non-2xx) response flips it back
    to `true`.
  - `getMutationStatusCounts`: a queued `pending` + a queued `failed` mutation for
    different entities produce independent `{pending, failed}` counts per entity.
