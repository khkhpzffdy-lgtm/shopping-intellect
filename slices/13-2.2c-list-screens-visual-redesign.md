GOAL

  `design/screens2.jsx` (the canonical mobile UI: app bar, list switcher, mode

  **visual-only** pass: restyle the existing two screens to match

  remove item, offline optimistic queue, mode toggle) keeps working exactly as
  before.

    relevant classes into `theme.css`/a new stylesheet) rather than re-deriving
    colours/spacing by hand. Check for class-name collisions with existing
    Tailwind/theme usage before wholesale importing.
  - **Lists overview** (`src/components/ListsScreen.tsx`):
    - Replace the current "Lists overview" card + plain input/button with the
      `ListCard` look from `design/screens2.jsx`/`10 §8.21` — name, item count,

    - Keep `SyncStatusIndicator` (`10 §8.19`) but restyle it to fit the new card
      (`design` doesn't show a dedicated sync badge on the overview yet — keep it
      subtle, e.g. small dot/label per `design/app.css` `.sync__dot`).
    - Create-list affordance: keep the existing input+button behaviour
      (`onCreateNameChange`/`onCreateList`), but restyle to match the design's
      app bar / FAB conventions as closely as practical — e.g. a "+" affordance
      that reveals the name input, per `design/screens2.jsx`'s `addbar` pattern.
    - Keep the empty state (`EmptyState`, `10 §8.26`) but restyle copy/visuals to
      match `design`'s tone (Bulgarian copy is fine to introduce here if it makes
      the restyle more faithful — confirm with existing app copy language first;

      spacing/colour, don't do a partial bilingual mix).
  - **List screen** (`src/components/ListScreen.tsx`):
    - App bar: title, back action (`onBack`), per `design/screens2.jsx`'s
      `appbar`/`appbar__title` (menu/search/kebab icons from the mockup are
      decorative placeholders for *this* slice — wire only `onBack`; the others
      may be inert/no-op buttons or omitted if they'd imply unbuilt features).
    - `ModeToggle` (`10 §8.18`): restyle to the `modeseg` segmented-control look
      (Планиране/Пазаруване or Planning/Shopping per existing app copy language),
      keeping the existing `onModeChange`/Zustand wiring from §2.2b.
    - Planning mode: restyle `ShoppingListItem` rows (`10 §8.1`) to the `git`
      emoji-row look — `term`, `qty`/`unit`, favorite heart, brand chip if
      `brand_anchor` set. **Emoji selection**: if no emoji-per-category data
      exists yet, use a single neutral placeholder emoji/icon for all items
      (don't invent a category→emoji mapping as product logic — that's a future
      slice; this is visual only).
    - Shopping mode: restyle to the large-checkbox `git--shop`/`git--done` look,
      tap toggles `is_checked` exactly as in §2.2b.
    - Add-item affordance: restyle to the `addbar` look (`+` / text input / mic
      icon — mic is decorative/inert, no speech-to-text in this slice).
    - FAB: the design's `fab` (scales icon, "compare prices") has **no backing
      feature yet** (comparison is M4). Either omit the FAB entirely for this
      slice, or render it disabled/inert with a tooltip — do **not** wire it to a
      stub navigation that dead-ends. Prefer omitting it.
    - Tab bar (`design`'s `TabBar`): the app currently has no other top-level
      screens to switch to (Profile, Add/Search as separate routes don't exist
      yet). Either omit the tab bar entirely, or render a single active "List"
      tab with the others inert — prefer omitting it. Don't build navigation to
      screens that don't exist.
